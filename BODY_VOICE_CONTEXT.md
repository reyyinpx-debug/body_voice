# 身体社交 · Body Voice 设计上下文

## 项目一句话
你的身体有自己的社交账号——Oura/Apple Watch 等可穿戴设备的每日数据，由 AI 以"身体本人"的口吻发社交动态。

## 当前技术状态
- **架构**: 纯前端（HTML/CSS/JS） + Node.js 后端（调用 AI API）
- **AI**: DeepSeek（`deepseek-chat`），服务端调用，API Key 不暴露给前端
- **数据源**: 90+ 天 Oura Ring 真实数据（每日睡眠/HRV/心率/HRV/步数/体温等）
- **部署**: 尚未部署，本地跑在 `http://localhost:3000`
- **文件**: `data-parser.js` 解析 Oura CSV；`api/generate.js` 调用 DeepSeek

## 用户流程
1. 上传 Oura CSV（或用示例数据）
2. 一键调 AI 批量生成所有日期的身体动态
3. 以 Feed 流展示（卡片式，每条带数据标签）

## 当前 Prompt（需要迭代）

当前在服务端 `api/generate.js` 和 `server.js` 中使用的 system prompt：

```
你是用户身体的意识。根据每天的生理数据，以"身体"的口吻写社交动态。

每条动态要求：
- 80-150字，中文口语化，拟人化
- 把数字变成叙事的一部分，不要直接罗列
- 如果有明显异常（比如睡眠少了、HRV高了）重点描写
- 每条独立，不要互相引用

你必须返回严格的JSON格式：
{"posts":[{"date":"日期","post":"动态正文"},...]}
```

### 用户反馈
第一条测试输出被评价为"太尴尬了"，需要大改。

## 待决策的 Prompt 方向

### 核心问题：身体的人格设定
需要确定"身体"到底是一个什么样的存在？

| 方向 | 风格 | 适合 |
|------|------|------|
| **俏皮室友** | 话多、拟声词、吐槽、开玩笑 | 年轻用户，小红书传播 |
| **温柔守护者** | 轻声细语、关心、用"咱们" | 健康/正念人群 |
| **禅意智者** | 简洁、哲理、像签文 | 东方美学爱好者 |
| **冷面数据员** | 冷静、客观、偶尔毒舌 | 极客/Quantified Self人群 |

### 语言风格选项
- 口语化程度：正式 ← → 极度口语化（带网络用语）
- 拟人化程度：客观描述 ← → 完整人格（有记忆、情绪、个性）
- 细节密度：简洁一句话 ← → 详细叙事
- 互动性：单向发帖 ← → 会@用户、提问、调侃

### 数据呈现方式
当前数据标签是独立展示的（"💤 睡眠 7h20m"），动态正文中不应重复罗列数字。问题是：正文中数字的"翻译"应该到什么程度？

- **轻度翻译**: "今晚睡得不错"（背后数据支撑但不提具体数字）
- **中度翻译**: "睡了整整7个小时，虽然不算多但质量很高"
- **重度叙事**: 把数字完全变成比喻/感受（"深睡占了一半，像沉到海底一样安静"）

### 异常处理
某天数据特别差/好时：
- 身体应该抱怨/庆祝吗？
- 还是保持中性描述？
- 还是把"锅"甩给用户（"你昨晚熬夜怪我咯"）？

## 后续可以做的事

### 多语气预设
用户可选 2-3 种语气（俏皮/温柔/冷静），system prompt 里加 `{tone}` 变量，前端下拉框切换。

### 身体签（Body Fortune）
每天一条"签"，融合数据 + 东方玄学/哲理，独立于 Feed 流展示。

### 对话功能
用户可以直接问身体"昨晚睡得好吗"，基于数据回答。
